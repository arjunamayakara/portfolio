AI-Powered Incident Response Platform
AI · Distributed Systems
GitHub → Deep Dive →

Built to close the gap between a monitoring alert firing and a remediation action being taken — without a human in the loop for failure patterns the system already knows. Grafana and PagerDuty tell you something is wrong. This platform figures out why and fixes it. The differentiator: team runbooks and past incident history are injected into the LLM prompt so it reasons with your operational knowledge, not generic SRE patterns.

Event-driven pipeline — 5,000+ metric events/min via Kafka (3 partitions, idempotent producer). Redis deduplication with 2-min window prevents alert storms.
Mistral LLM via LangChain4j diagnoses root cause from structured prompt. Runbooks and past incidents injected as context on every call — context-aware AI, not generic advice.
Auto-remediates: SCALE_UP, RESTART_POD, ROLLBACK, ALERT_TEAM, INVESTIGATE. Real Slack notifications with AI root cause embedded in every alert.
Prometheus Alertmanager webhook integration — connecting to a real monitoring stack is a config change, not a code change.
Same codebase ships as a single JAR (monolith) or 4 independent microservices. No code changes to switch.
LLM provider abstracted via LangChain4j — swap Mistral for GPT-4 in one line of config. Everything else stays unchanged.
